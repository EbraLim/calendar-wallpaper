# CLAUDE.md

이 파일은 Claude Code가 이 레포에서 작업할 때 따라야 할 규칙이다. 매 세션 먼저 읽는다.

## 프로젝트
사진 + 그 달 달력을 합쳐, 시계·알림·피사체에 가리지 않는 위치에 달력을 얹은
잠금화면 이미지를 만드는 **모바일 웹앱(PWA)**. 그 달 기준 **정적 이미지**를 생성한다.
제품 정의는 `SPEC.md` 참고.

## 기술 스택 (결정됨, 함부로 바꾸지 말 것)
- 빌드: **Vite + TypeScript** (프레임워크 없는 바닐라로 시작 — 검토하기 쉽게)
- 렌더링: **HTML Canvas** (이미지 + 달력 그리기 + 다운로드)
- PWA: **1일차부터** manifest + service worker
- 배포: Vercel 또는 Netlify (초기에 라이브 URL 확보)
- 외부 의존성 최소화. 큰 라이브러리 넣기 전엔 먼저 물어볼 것.

## 절대 규칙 (Hard rules)
1. **이미지당 유료 API 호출 금지.** v1의 배치는 기하학 + 사용자 드래그로만 푼다. 단가 파괴 = 프로젝트 사망.
2. **이미지는 가능한 한 클라이언트에서만 처리.** 사용자 사진을 서버로 안 보내는 걸 기본값으로(프라이버시 + 비용).
3. **PWA 구조를 처음부터.** 나중에 재설계하지 않도록 manifest/service worker를 초기에 깐다.
4. 큰 변경은 **작은 diff로 쪼개서** 제안하고, 매 슬라이스 후 멈춰서 확인받는다. 수백 줄을 한 번에 쏟지 말 것.
5. 새 의존성 추가, 스택 변경, 폴더 구조 대수술 전에는 **먼저 묻는다.**

## 핵심 도메인 주의사항 (버그 단골 지점)
- **달력 날짜 로직**: 월 시작 요일, 말일, 월 경계 off-by-one을 항상 의심. 예) 6월 달력에 7월 날짜가 새거나 요일이 한 칸 밀리는 것. → 이 로직엔 **단위 테스트를 반드시** 작성.
- **시스템 UI 안전지대**: 시계/날짜/알림 위치는 탐지 대상이 아니라 **OS별 상수**다. `src/safezones.ts` 같은 데이터로 정의하고, 달력이 그 영역을 피하게 한다.
- **기기 종횡비**: 정확한 기기 모델은 못 읽는다. **종횡비 프리셋**(예: iPhone Pro, 흔한 Android 비율)을 제공하고, 출력은 타깃 해상도로 렌더한다.
- **iOS 크롭/줌**: iOS 잠금화면은 사진을 살짝 줌/크롭하므로 가장자리에 **여유 마진**을 둔다.
- **한국 특화**(후순위 슬라이스): 음력 표기, 한국 공휴일, 한글 타이포. 라이브러리/데이터는 도입 전에 상의.

## 빌드 순서 (위험·가치 우선, 슬라이스 단위)
1. 스택 셋업 + 이 파일 정비 + 초기 배포로 라이브 URL 확보
2. **코어 렌더링**: 고정 이미지 1장 + 기기 1개 → Canvas에 달력 오버레이 → 다운로드 (가장 먼저 증명)
3. 이미지 업로드 + 종횡비 프리셋
4. 안전지대 기본 배치 + **드래그로 재배치**
5. 폴리시: 달력 스타일(색/폰트/투명도), 사진 밝기에 따른 자동 대비
6. 한국 특화: 공휴일 / 음력
7. PWA 설치 가능화 + 매니페스트 마무리
8. 재유입: 월간 푸시 (Android 무료 / iOS는 설치된 PWA 한정)
> v2(별도): 클라이언트사이드 얼굴·세일런시 탐지로 자동 배치 "추천"

## 작업 방식
- 슬라이스마다 **실제 폰에서 테스트** 가능한 상태로 끝낸다.
- 슬라이스 단위로 커밋. 커밋 메시지는 무엇을/왜.
- 날짜·안전지대 같은 핵심 로직은 테스트 먼저 또는 함께.
- 구조나 결정이 바뀌면 이 `CLAUDE.md`를 업데이트한다.

## 하지 말 것
- iOS에서 배경화면을 코드로 설정하려 시도하지 말 것 (불가능). 마지막은 사용자가 사진앱에서 수동 설정 — 그 안내 UX를 잘 만들 것.
- 달력을 자동 갱신하는 라이브 월페이퍼/백그라운드 작업 만들지 말 것 (v1 범위 밖).
- 검토 안 된 대규모 코드 한꺼번에 생성 금지.
